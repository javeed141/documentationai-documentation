### Ruby

Generate idiomatic Ruby code using the `net/http` standard library:
````ruby
require 'net/http'
require 'json'
require 'uri'

# POST request with JSON body
uri = URI('/api/users')
http = Net::HTTP.new(uri.host, uri.port)
http.use_ssl = true

request = Net::HTTP::Post.new(uri)
request['Authorization'] = 'Bearer YOUR_TOKEN'
request['Content-Type'] = 'application/json'
request.body = {
  name: 'John Doe',
  email: 'john@example.com'
}.to_json

response = http.request(request)
raise "HTTP #{response.code}: #{response.message}" unless response.is_a?(Net::HTTPSuccess)

user = JSON.parse(response.body)
puts "Created user: #{user['id']}"
````

Features include:
- **Standard library `net/http`** with no external dependencies
- **JSON serialization** using built-in `json` module
- **Error handling** with HTTP response class checking
- **Ruby conventions** including symbols, string interpolation, and snake_case naming


## Prerequisites

Before you begin, ensure you have:

- A modern web browser (Chrome, Firefox, Safari, or Edge)

- An email address for account registration

- Internet connection

<Callout kind="info" collapsed="false">
  Catapillar works entirely in the browser—no downloads required. All you need is a free account to start creating documentation spaces.
</Callout>

## Step 1: Create Your Account

Sign up for Catapillar in under a minute.

<Steps>
  <Step title="Visit the Dashboard" icon="globe" title-type="p">
    Navigate to `https://dashboard.catapillar.com` and click **Sign Up**.
  </Step>

  <Step title="Enter Your Details" icon="user" title-type="p">
    Provide your email, create a password, and complete the CAPTCHA.
  </Step>

  <Step title="Verify Email" icon="mail" title-type="p">
    Check your inbox for a verification link from Catapillar and click it to activate your account.
  </Step>
</Steps>

## Step 2: Log In and Create a Documentation Space

Once logged in, create your first space.

<Tabs>
  <Tab title="From Dashboard" icon="home">
    <Steps>
      <Step title="New Space" icon="plus" title-type="p">
        Click **New Documentation Space** on the dashboard.
      </Step>

      <Step title="Name Your Space" icon="edit" title-type="p">
        Enter a name like `My Project Docs` and select a visibility (public or private).
      </Step>

      <Step title="Customize" icon="settings" title-type="p">
        Add a description and choose your brand color (`#3B82F6` for Catapillar blue).
      </Step>
    </Steps>
  </Tab>

  <Tab title="Quick Import" icon="upload">
    Use the import wizard for GitHub repos or Markdown files.
  </Tab>
</Tabs>

## Step 3: Add and Edit Your First Document

Publish your first page instantly.

<Steps>
  <Step title="New Document" icon="file-plus" title-type="p">
    In your space, click **New Page** and title it `Introduction`.
  </Step>

  <Step title="Edit Content" icon="edit-3" title-type="p">
    Use the visual editor or switch to Markdown mode. Here's a simple MDX example:

    <CodeGroup show-lines="true" tabs={[]}>
      ````markdown
      # Welcome to My Docs

      This is your first document.

      ```javascript
      console.log("Hello, Catapillar!");
      ```
      ````

      ````mdx
      ## Rendered Preview

      **Bold text** and `inline code`.

      ```javascript
      console.log("Hello, Catapillar!");
      ```
      ````
    </CodeGroup>
  </Step>

  <Step title="Publish" icon="upload-cloud" title-type="p">
    Click **Publish** to make it live. Share the URL: `https://docs.catapillar.com/your-space/introduction`.
  </Step>
</Steps>

<Callout kind="tip" collapsed="false">
  Use the preview pane to see changes in real-time. MDX supports components like `<Callout>` for rich docs.
</Callout>
