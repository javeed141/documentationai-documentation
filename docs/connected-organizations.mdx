title: Connected Organizations
---

A Connected Organization setup in Capillary is designed for conglomerates where each brand is created as an independent organization. These brand organizations, referred to as **child organizations**, are linked to a central **parent organization**.

<Image src="https://files.readme.io/a3f6fdef6684900c6e502680c0ea73e97fba544fe75a225ea786fc6969c19c0d-Conencted_Org_Final.png" />

**Child organization**

In a connected organization, each child org works independently, with its own data, customer attributes, loyalty program, and engagement initiatives. Child orgs do not have access to each other’s data, ensuring strict boundaries and data security.

**Parent organization**

The parent org acts as the central authority. It has visibility into customer profiles and events across connected child orgs. When a customer registers in a child org, the profile is first created at the parent org and then made available in the child org. The parent org can run group-level loyalty programs and marketing operations, while child orgs can continue running their own independent loyalty and campaign operations.

**Overall benefit**

Since each brand operates within its own organization, brands retain full control over their customer data and operations, without cross-organizational access. At the same time, the parent org will have a unified overview and can run umbrella-level loyalty programs.

## Creating a connected organization

To create a connected organization setup, you need to first create a parent organization, followed by the individual child organizations within the same cluster. Then, raise a JIRA ticket to the Product Support team to map the child organizations to the parent organization.
