---
title: Push to S3
---

Configuration reference for the Push to S3 block — uploads the processed output file to an AWS S3 bucket.

The Push to S3 block transfers the processed output file to an AWS S3 bucket. It is used in templates such as FTP to S3 and the Audience Reload from SFTP template.

**Block type:** `PutS3ObjectV2`

Uploads the output file to a specified S3 bucket using AWS credentials. Supports all file types via the `application/octet-stream` content type.

## Configuration fields

| Field                      | Description                                                                                                                                           |
| :------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------- |
| S3 Object name/Filename * | The target key (path + filename) for the object in S3. For audience templates, use the format `connect_plus_audience_upload/<group_name>/${filename}` |
| S3 Bucket Name *          | Name of the target S3 bucket                                                                                                                          |
| Access Key *              | AWS access key ID                                                                                                                                     |
| Secret Key *              | AWS secret access key                                                                                                                                 |
| AWS Region *              | AWS region of the target bucket                                                                                                                       |
| Content Type               | MIME type for the uploaded file. Default: `application/octet-stream`                                                                                  |

* Required field
