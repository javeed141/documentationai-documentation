---
title: Reward Tracking with Status and Custom Fields in Reward Fulfillment
---

You can utilize the standard field "Status" and add custom fields in the issueReward API to include additional metadata related to reward issuance for tracking purposes. This is especially useful for tracking rewards fulfilled outside the Capillary system.

<p><strong><span>Key Features:</span></strong></p>

<p><span>Standard Field - STATUS:</span></p>

<p><span>A new ENUM field called STATUS is now available in the issueReward call to track the current status of reward fulfillment (e.g., received, in-progress, pending, denied, completed).</span></p>

<p><span>This field is optional and validated against predefined values.</span></p>

<p><span>Custom Fields:</span></p>

<p><span>Issue Reward call now supports custom fields with data types such as String, Integer, Date, and Boolean.</span></p>

<p><span>Custom fields are unique at the organization level, with options for default values and case sensitivity.</span></p>

<p><span>A max of 10 custom fields can be passed in the issue Reward call</span></p>

<p><span>Integration with Other APIs:</span></p>

<p><span>The newly introduced fields are supported by the getPurchasedRewards API and membercare API, ensuring consistent data representation across platforms.</span></p>

<p><strong><span>Benefits:</span></strong></p>

<p><span>Enhanced Metadata Handling : Improved tracking of reward fulfillment statuses and additional metadata to streamline operations.</span></p>

<p><span>Improved Customer Interactions :Real-time updates and detailed information available on partner portals and mobile apps, enhancing customer satisfaction.</span></p>

<p><strong><span>Example Use Case:</span></strong></p>

<p><span>Fulfillment Process:</span></p>

<p><span>A reward partner logs into the partner portal and sees a list of rewards with a pending STATUS.</span></p>

<p><span>Upon receiving a fulfillment request, the partner updates the STATUS to confirmed.</span></p>

<p><span>Additional metadata such as ticket number or order ID can be entered.</span></p>

<p><span>Once fulfillment is complete, the STATUS is updated to completed.</span></p>
