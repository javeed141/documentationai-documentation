title: View Event Feed

The **Event feed** page displays a log of all events generated within the organisation for all events over the last **30 days**. Use this view to track event activity, monitor system behaviour, or troubleshoot event-based processes.

You can filter events by **customer**, **event type**, or **date range** to quickly locate specific records.

<Image src="https://files.readme.io/62e3fe0a66f98189d054924c13fca7b6659115996128eaec331b8cf932c4bdee-image.png" />

| Field | Description |
| --- | --- |
| Search and filter | Lets you: * search for specific events by customer identifier (email, mobile number, or external ID), card number, transaction/bill number, and API request ID (you can find this from the API reponse header). * filter by event dates * filter by event type such as transactionAdd, customerAdd etc * filter by event status such as pending, delivered, failed, and retrying |
| Event occurred at | Displays the exact timestamp when the event was triggered in the Capillary system in organization timezone. |
| Event type | Indicates the type of event that was sent to the webhook (for example, `customerUpdated`, `transactionAdded`, `pointsIssued`) along with the event ID. The event ID is auto-generated whenever an event is generated. |
| API request ID | Displays the unique identifier for each API request, which can be used for tracking or troubleshooting webhook deliveries. |

***

<Callout kind="info">

  Filter usage note

  You can **filter by either event type or search option**, but not both at the same time.
  If you use the search option to look up a specific customer or identifier, the event type filter becomes inactive, and vice versa.

</Callout>

## View the event payload

When you open an event from the **Event feed**, the **Event details** view displays complete information about that specific event. You can use this view to inspect the event payload received by the Event Notification Service (ENS).

<Image src="https://files.readme.io/eb16af587717bc60eece0988b7a2836df3ebbbe07aef9791504a3bc1a6537472-Event_Feed.gif" />

### Overview section

The **Overview** section summarises key identifiers and timestamps for the event.

<Image src="https://files.readme.io/896a840e2825562fc97e793996918b57369dca6f34479ccf0e8aff035c8a215f-image.png" />

| Field             | Description                                                                                                                                      |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| Event ID          | A unique identifier for the event generated whenever an event is triggered.                                                                      |
| Event occurred at | The exact timestamp when the event was generated in the Capillary system in organization timezone.                                               |
| API request ID    | The unique request identifier associated with the API call that triggered the event. You can find the API request ID in the API response header. |

***

### Event payload

The **Event payload** section displays the actual JSON data for the selected event. You can toggle between **View original payload** and **enriched payload** (if payload enrichment is enabled) to compare the raw event data with the additional attributes appended by ENS.

Use the **copy** icon to quickly copy the payload for debugging, validation, or integration testing.
