---
title: Operators
---

**Operators** are the logic functions used to compare attributes against values when building qualifying conditions. They determine how the system checks a specific data point. The available operators are based on the data type of the attribute you are evaluating.

<Expandable title="1. Standard Comparison Operators">

  *Applicable to Strings, Dates, and Numbers.*

  | Operator                        | Data Type               | Description                                                        | Examples                                                                                                                                                                                             |
  | :------------------------------ | :---------------------- | :----------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | **Equals**                      | String, Date, Number    | Checks whether the attribute matches the value exactly.            | Example 1: `Member Attributes` &gt; `First Name` &gt; `Equals` &gt; `John` <br /> Example 2: `Purchase attributes` &gt; `Gross Amount` &gt; `Equals` &gt; `100.50`                                                     |
  | **Not Equals**                  | String, Date, Number    | Checks whether the attribute isn't equal to the specified value.   | Example 1: `Member Attributes` &gt; `Slab Name` &gt; `Not Equals` &gt; `Gold` <br /> Example 2: `Store Attributes` &gt; `Name` &gt; `Not Equals` &gt; `Main Street`                                                    |
  | **Is Null**                     | String, Date, Number    | Checks whether the field is empty (contains no data).              | Example 1: `Member Attributes` &gt; `Email` &gt; `Is Null` <br /> Example 2: `Purchase attributes` &gt; `Notes` &gt; `Is Null`                                                                                   |
  | **Is Not Null**                 | String, Date, Number    | Checks whether the field contains any value.                       | Example 1: `Member Attributes` &gt; `Mobile` &gt; `Is Not Null` <br /> Example 2: `Card Attributes` &gt; `Card Number` &gt; `Is Not Null`                                                                        |
  | **Is One Of**                   | String, List of Strings | Matches any single item in the list.                               | Example 1: `Member Attributes` &gt; `Tier` &gt; `Is One Of` &gt; `Gold, Platinum` <br /> Example 2: `Purchase attributes` &gt; `Tender Code` &gt; `Is One Of` &gt; `CASH, VISA`                                        |
  | **Is None Of**                  | String, List of Strings | Matches only if the value isn't in the list.                       | Example 1: `Member Attributes` &gt; `Status` &gt; `Is None Of` &gt; `Blocked, Fraud` <br /> Example 2: `Store Attributes` &gt; `Code` &gt; `Is None Of` &gt; `Store_001`                                               |
  | **Previous Value**              | String                  | Checks what the value was before the change.                       | Example 1: `Member Update attributes` &gt; `Mobile` &gt; `Previous Value` &gt; `9999999999` <br /> Example 2: `Member Update attributes` &gt; `Email` &gt; `Previous Value` &gt; `old@mail.com`                        |
  | **Previous Custom Field Value** | String                  | Checks the value of a specific custom field before it was changed. | Example 1: `Member Update attributes` &gt; `Custom Field` &gt; `Previous Custom Field Value` &gt; `Standard` <br /> Example 2: `Member Update attributes` &gt; `Tier` &gt; `Previous Custom Field Value` &gt; `Silver` |
  | **Current Custom Field Value**  | String                  | Checks the value of a specific custom field after it has changed.  | Example 1: `Member Update attributes` &gt; `Custom Field` &gt; `Current Custom Field Value` &gt; `Premium` <br /> Example 2: `Member Update attributes` &gt; `Status` &gt; `Current Custom Field Value` &gt; `Active`  |
  | **Current Value**               | String                  | Checks the value after the change.                                 | Example 1: `Member Update attributes` &gt; `Last Name` &gt; `Current Value` &gt; `Smith`  Example 2: `Member Update attributes` &gt; `Loyalty Type` &gt; `Current Value` &gt; `Loyal`                                  |
  | **Exists**                      | String, List of Strings | Checks whether the attribute key is in the data.                   | Example 1: `Member Attributes` &gt; `External Id` &gt; `Exists` <br /> Example 2: `Purchase attributes` &gt; `Custom Field` &gt; `Exists`                                                                        |
  | **Not Exists**                  | String, List of Strings | Checks whether the attribute key is not included in the data.      | Example 1: `Member Attributes` &gt; `Partner ID` &gt; `Not Exists` <br /> Example 2: `Card Attributes` &gt; `Legacy ID` &gt; `Not Exists`                                                                        |

</Expandable>

<Expandable title="2. Text and List Operators">

  *Specific to String fields and List of Strings.*

  | Operator           | Data Type               | Description                                                             | Examples                                                                                                                                                                          |
  | :----------------- | :---------------------- | :---------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | **Contains**       | String, List of Strings | Checks whether the text includes a specific sequence of letters.        | Example 1: `Member Attributes` &gt; `Full Name` &gt; `Contains` &gt; `Smith` <br /> Example 2: `Purchase attributes` &gt; `Item Description` &gt; `Contains` &gt; `Organic`                         |
  | **Starts With**    | List of Strings         | Checks whether the value begins with a specific prefix.                 | Example 1: `Purchase attributes` &gt; `Item Code` &gt; `Starts With` &gt; `SKU` <br /> Example 2: `Member Attributes` &gt; `Mobile` &gt; `Starts With` &gt; `+1`                                    |
  | **Is Empty**       | String                  | Checks whether the field is empty (contains no data).                   | Example 1: `Member Attributes` &gt; `Middle Name` &gt; `Is Empty` <br /> Example 2: `Purchase attributes` &gt; `Tender Notes` &gt; `Is Empty`                                                 |
  | **Matches**        | String, List of Strings | Checks whether the value matches a specific regular expression pattern. | Example 1: `Member Attributes` &gt; `First Name` &gt; `Matches` &gt; `^[A-Z]` <br /> Example 2: `Card Attributes` &gt; `Vehicle Number` &gt; `Matches` &gt; `[0-9]{4}`                              |
  | **Matches Any Of** | List of Strings         | Checks whether at least one item in a list matches the criteria.        | Example 1: `Purchase attributes` &gt; `Item Code` &gt; `Matches Any Of` &gt; `Milk, Bread` <br /> Example 2: `Purchase attributes` &gt; `Tender Code` &gt; `Matches Any Of` &gt; `VISA, AMEX`       |
  | **Matches All Of** | List of Strings         | Checks whether every item in the list matches the criteria.             | Example 1: `Purchase attributes` &gt; `Item Code` &gt; `Matches All Of` &gt; `ItemA, ItemB` <br /> Example 2: `Purchase attributes` &gt; `Promotion Code` &gt; `Matches All Of` &gt; `SALE10, APP5` |
  | **Not In**         | List of Strings         | Checks whether the specific value isn't in the list.                    | Example 1: `Purchase attributes` &gt; `Item Code` &gt; `Not In` &gt; `Restricted_01` <br /> Example 2: `Store Attributes` &gt; `Code` &gt; `Not In` &gt; `Online_Store`                             |
  | **Is Valid Date**  | String                  | Checks whether a text string is in a valid date format.                 | Example 1: `Member Attributes` &gt; `Passport Expiry String` &gt; `Is Valid Date` <br /> Example 2: `Card Attributes` &gt; `Activation String` &gt; `Is Valid Date`                           |

</Expandable>

<Expandable title="3. Numerical Operators">

  *Specific to Real numbers, Integers, and List of Numbers.*

  | Operator                  | Data Type               | Description                                                                                        | Examples                                                                                                                                                             |
  | :------------------------ | :---------------------- | :------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | **Greater Than**          | Number, List of Numbers | Checks whether the value is greater than the specified value.                                      | Example 1: `Member Attributes` &gt; `Current Points` &gt; `Greater Than` &gt; `1000` <br /> Example 2: `Purchase attributes` &gt; `Gross Amount` &gt; `Greater Than` &gt; `50`         |
  | **Less Than**             | Number, List of Numbers | Checks whether the value is less than the specified value.                                         | Example 1: `Member Attributes` &gt; `Slab Number` &gt; `Less Than` &gt; `3` <br /> Example 2: `Member Attributes` &gt; `Number of Visits` &gt; `Less Than` &gt; `5`                    |
  | **Greater Than Or Equal** | Number, List of Numbers | Checks whether the value is Checks whether the value is less than or equal to the specified value. | Example 1: `Member Attributes` &gt; `Age` &gt; `Greater Than Or Equal` &gt; `18` <br /> Example 2: `Purchase attributes` &gt; `Item Quantity` &gt; `Greater Than Or Equal` &gt; `2`    |
  | **Less Than Or Equal**    | Number, List of Numbers | Checks whether the value is less than or equal to the specified value.                             | Example 1: `Member Attributes` &gt; `Days Since Visit` &gt; `Less Than Or Equal` &gt; `30` <br /> Example 2: `Purchase attributes` &gt; `Discount` &gt; `Less Than Or Equal` &gt; `20` |
  | **Is Between**            | Number, List of Numbers | Checks whether the value falls within a range (inclusive).                                         | Example 1: `Member Attributes` &gt; `Age` &gt; `Is Between` &gt; `21, 35` <br /> Example 2: `Purchase attributes` &gt; `Run Time` &gt; `Is Between` &gt; `60, 120`                     |
  | **Interval**              | Number                  | Checks whether the number matches a specific interval.                                             | Example 1: `Member Attributes` &gt; `Number of Visits` &gt; `Interval` &gt; `5` <br /> Example 2: `Purchase attributes` &gt; `Gross Amount` &gt; `Interval` &gt; `100`                 |
  | **Year**                  | Number                  | Extracts the year from a date for numeric comparison.                                              | Example 1: `Purchase attributes` &gt; `Date` &gt; `Year` &gt; `Equals` &gt; `2025` <br /> Example 2: `Member Attributes` &gt; `Birth Date` &gt; `Year` &gt; `Less Than` &gt; `2000`          |
  | **Month**                 | Number                  | Extracts the month (1-12) from a date.                                                             | Example 1: `Purchase attributes` &gt; `Date` &gt; `Month` &gt; `Equals` &gt; `12` <br /> Example 2: `Member Attributes` &gt; `Enrolment Date` &gt; `Month` &gt; `Is One Of` &gt; `6, 7, 8`   |
  | **Day**                   | Number                  | Extracts the day of the week or the numeric day.                                                   | Example 1: `Purchase attributes` &gt; `Date` &gt; `Day` &gt; `Equals` &gt; `15` <br /> Example 2: `Member Attributes` &gt; `Join Date` &gt; `Day` &gt; `Is Not` &gt; `1`                     |

</Expandable>

<Expandable title="4. Date Operators">

  *Specific to Date fields.*

  | Operator            | Data Type | Description                                                                     | Examples                                                                                                                                                                        |
  | :------------------ | :-------- | :------------------------------------------------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
  | **Is**              | Date      | Checks whether the date falls on a specific unit, for example, weekend).        | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is` &gt; `Weekend` <br /> Example 2: `Purchase attributes` &gt; `Date` &gt; `Is` &gt; `Day`                                                    |
  | **Is Not**          | Date      | Checks whether the date doesn't fall on a specific unit.                        | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is Not` &gt; `Weekend` <br /> Example 2: `Purchase attributes` &gt; `Date` &gt; `Is Not` &gt; `Weekday`                                        |
  | **Is Before**       | Date      | Checks whether the date is earlier.                                             | Example 1: `Member Attributes` &gt; `Join Date` &gt; `Is Before` &gt; `2024-01-01` <br /> Example 2: `Card Attributes` &gt; `Expiry Date` &gt; `Is Before` &gt; `2025-12-31`                      |
  | **Is After**        | Date      | Checks whether the date is later.                                               | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is After` &gt; `2024-01-01` <br /> Example 2: `Member Attributes` &gt; `Last Login` &gt; `Is After` &gt; `2025-02-01`                          |
  | **Is On Or Before** | Date      | Checks whether the date is on or before a specified date.                       | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is On Or Before` &gt; `2024-12-31` <br /> Example 2: `Member Attributes` &gt; `Slab Expiry` &gt; `Is On Or Before` &gt; `2025-06-30`           |
  | **Is On Or After**  | Date      | Checks whether the date is on or after a specified date.                        | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is On Or After` &gt; `2025-01-01` <br /> Example 2: `Card Attributes` &gt; `Start Date` &gt; `Is On Or After` &gt; `2023-01-01`                |
  | **Is Time Between** | Date      | Checks whether the time is between the specified times.                         | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is Time Between` &gt; `12:00, 14:00` <br /> Example 2: `Purchase attributes` &gt; `Date` &gt; `Is Time Between` &gt; `06:00, 09:00`            |
  | **Is Hour Between** | Date      | Checks whether the hour is from 0 through 23.                                   | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is Hour Between` &gt; `12, 14` <br /> Example 2: `Purchase attributes` &gt; `Date` &gt; `Is Hour Between` &gt; `17, 22`                        |
  | **Is Valid**        | Date      | Checks whether the date is a valid date object.                                 | Example 1: `Member Attributes` &gt; `Birth Date` &gt; `Is Valid` <br /> Example 2: `Card Attributes` &gt; `Activation Date` &gt; `Is Valid`                                                 |
  | **Day Of Month**    | Date      | Checks whether the day of the month is from 1 through 31.                       | Example 1: `Purchase attributes` &gt; `Date` &gt; `Day Of Month` &gt; `1` <br /> Example 2: `Member Attributes` &gt; `Join Date` &gt; `Day Of Month` &gt; `15`                                    |
  | **Days Diff**       | Date      | Calculates the count of days between two dates.                                 | Example 1: `Purchase attributes` &gt; `Date` &gt; `Days Diff` &gt; `Member Attributes.Join Date` <br /> Example 2: `Card Attributes` &gt; `Expiry Date` &gt; `Days Diff` &gt; `Current Date`      |
  | **Date Diff**       | Date      | Calculates the absolute difference between two dates.                           | Example 1: `Purchase attributes` &gt; `Date` &gt; `Date Diff` &gt; `Member Attributes.Join Date`  Example 2: `Redemption attributes` &gt; `Date` &gt; `Date Diff` &gt; `Purchase attributes.Date` |
  | **Minutes Diff**    | Date      | Calculates the number of minutes between two dates.                             | Example 1: `Purchase attributes` &gt; `Date` &gt; `Minutes Diff` &gt; `30` <br /> Example 2: `Purchase attributes` &gt; `Show Time` &gt; `Minutes Diff` &gt; `15`                                 |
  | **Is Before (TZ)**  | Date      | Checks whether a date is earlier than a specified date in a specific time zone. | Example 1: `Purchase attributes` &gt; `Date` &gt; `Is Before With Timezone` &gt; `Asia/Kolkata` <br /> Example 2: `Member Attributes` &gt; `Event Date` &gt; `Is Before With Timezone` &gt; `GMT` |

</Expandable>

  | **Partner Program Update Type** | String (Enum) | Checks the reason for a tier change (Upgrade, Downgrade, or Renew).          | Example 1: `Partner attributes` &gt; `Update Type` &gt; `Equals` &gt; `UPGRADE` <br /> Example 2: `Partner attributes` &gt; `Update Type` &gt; `Is None Of` &gt; `DOWNGRADE`                    |

</Expandable>
