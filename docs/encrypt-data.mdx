Configuration reference for the Encrypt Data block — encrypts the output file before it is transferred to the destination.

The Encrypt Data block encrypts the output file before it is transferred to the destination. It is placed between the last processing block and the Connect to Destination block.

**Block type:** `optional_encrypt_content` / `DynamicEncryptContent`

Encrypts the processed file using the specified algorithm and key. Supports PGP (asymmetric) and AES (symmetric) encryption.

## Configuration fields

| Field                          | Description                                                                                     |
| :----------------------------- | :---------------------------------------------------------------------------------------------- |
| Encryption Method *           | Encryption algorithm: `PGP`, `AES-256`, `AES-128`. PGP uses a public key; AES uses a passphrase |
| Encryption Key / Public Key * | The PGP public key or AES passphrase used for encryption                                        |
| Is Enabled                     | Toggle to enable or disable encryption without removing the block                               |

* Required field
