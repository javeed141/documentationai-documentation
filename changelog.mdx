## Upgrade Instructions

Follow these steps to upgrade Catapillar safely.

<Steps>
  <Step title="Backup Data" icon="database">
    Export your projects from `https://dashboard.example.com/export`.
  </Step>
  <Step title="Update Package" icon="package">
    <CodeGroup tabs="npm,yarn">
````bash
npm install catapillar@latest
````
````bash
yarn add catapillar@latest
````
    </CodeGroup>
  </Step>
  <Step title="Run Migrations" icon="git-branch">
    ```bash
    npx catapillar migrate
    ```
  </Step>
  <Step title="Restart Service" icon="refresh-cw">
    Restart your application server.
  </Step>
</Steps>

<Callout kind="tip">
  Test upgrades in staging first. Use `catapillar --dry-run` to preview changes.
</Callout>

## Migration Guide

<ExpandableGroup>
  <Expandable title="v1.0 Breaking Changes" default-open="true">
    The API now requires `{API_KEY}` headers instead of query params.

    ```javascript
    // Old
    fetch('https://api.example.com/v0/projects?api_key=YOUR_API_KEY')

    // New
    fetch('https://api.example.com/v1/projects', {
      headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` }
    })
    ```

    Update `<ParamField header="Authorization" param-type="string" required="true">Bearer token</ParamField>` in your requests.
  </Expandable>
  <Expandable title="Deprecated Features">
    Legacy file uploads end support in v2.0. Migrate to the new multipart API.
  </Expandable>
</ExpandableGroup>
